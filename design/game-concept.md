# 花札在线对战

## 一句话描述

> 一款支持多人实时在线对战的日本花札小游戏（微信小游戏），包含 Koi-Koi 和 Hachi-Hachi 两种经典规则变体。

## 类型与平台

- **类型**：卡牌策略 + 休闲对战
- **平台**：微信小游戏
- **视角**：2D 俯视牌桌

## 目标受众

- **核心玩家**：喜欢花札/桌游的玩家、日本文化爱好者、休闲卡牌玩家
- **Bartle 类型倾向**：Achievers（收集成就）+ Socializers（社交对战）
- **年龄评级**：E（全年龄）

## 核心循环

```
短期（每回合）：看手牌 → 出牌 → 翻牌 → 判定是否配对 → 收集/得分
中期（每局）：累积点数 → 选择喊停(こいこい)继续 or 结算
长期（多局）：提升排名 → 解锁牌面/牌桌皮肤 → 社交互动
```

### 循环时间

- **短期循环**（秒级）：出牌 → 翻牌 → 收集，单次操作 3-10 秒
- **中期循环**（分钟级）：一局花札约 5-15 分钟
- **长期循环**（天级）：日常对战、排行榜、成就

## 游戏支柱 (Pillars)

1. **规则正统**：严格遵循传统 Koi-Koi 和 Hachi-Hachi 规则，让玩家体验到正宗花札
2. **上手简单、精通有深度**：新玩家 30 秒学会基本操作，老玩家可以研究策略
3. **社交乐趣**：实时对战的紧张感 + 轻松休闲的氛围
4. **一人可开发**：所有设计考虑个人开发者的时间和资源约束

## 核心机制

### 花札基本玩法

- **48 张牌**，12 个月份各 4 张，每张牌属于一个类别（光、种、短册、カス）
- **匹配机制**：出的牌和场上翻出的牌同月份则配对收集
- **得分系统**：凑齐特定组合（役）获得点数

### Koi-Koi 模式

- 两人对战
- 凑出役后可以选择「こいこい」（继续赌更大）或结算
- 风险与收益的博弈

### Hachi-Hachi 模式

- 三人对战
- 规则略有不同，更注重策略

## 独特卖点 (USP)

1. **微信内即点即玩** — 无需下载 App，分享给好友直接开打
2. **两种经典规则** — Koi-Koi + Hachi-Hachi 都支持
3. **2-3 人灵活对战** — 不局限于传统两人
4. **实时对战的社交体验** — 看着对手思考的紧张感

## 竞品分析

| 游戏 | 相似点 | 差异点 | 可借鉴之处 |
|------|--------|--------|------------|
| Hanafuda Do (手游) | 花札玩法 | 单人/离线为主 | UI 设计、牌面美术 |
| 各种小程序花札 | 小程序平台 | 规则不全、无实时对战 | 小程序交互模式 |

## 范围与约束

- **团队规模**：1 人
- **开发周期**：MVP 预计 4-6 周
- **技术约束**：微信小游戏平台限制（包体 ≤ 4MB 主包、需适配微信 API）
- **内容约束**：牌面美术使用开源/免版权素材，可后续替换
- **预算约束**：低成本个人项目

## 成功标准

- **MVP 标准**：Koi-Koi 两人实时对战可玩，基础匹配 + 结算正常
- **发布标准**：两种模式都支持，稳定运行，无明显 Bug
- **商业目标**：初期以验证玩法为主

## 风险与假设

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| 微信小游戏实时对战技术复杂度 | 高 | 先做原型验证 WebSocket 连通性 |
| 花札规则实现复杂度 | 中 | 严格参考规则文档，逐模块实现 |
| 3 人模式逻辑复杂 | 中 | 先做好 2 人模式，3 人作为扩展 |
| 牌面素材版权 | 低 | 使用明确可商用的素材 |
