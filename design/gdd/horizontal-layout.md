# 横屏布局改造 — 游戏设计文档

> 创建日期：2026-05-06
> 需求来源：用户要求
> 核心原则：**玩法不变，只改 UI 布局**
> 视觉稿确认日期：2026-05-06 ✅

---

## 概述

- **改造目标**：将现有竖屏花牌游戏改为横屏布局，参考麻将桌的视觉风格
- **玩法变更**：无（保持 Koi-Koi / Hachi-Hachi 原有规则）
- **优先级**：P1

### 计分区设计要点

1. **必须展示得分明细**：每个役组合单独列出，带注释说明
2. **展开/收起**：默认收起显示合计，点击展开看明细
3. **役组合预览**：显示组成该役的具体牌（缩略图，轻微重叠）
4. **未成役提示**：显示差几张成役
5. **Koi 加成**：单独显示倍数和最终得分
6. **月份分组**（可选）：按 12 个月份分组展示收集的牌（参考图布局）

### 缩略图绘制规范

```
麻将风格摆牌：

│🎴 │ │🎴 │ │  ← 轻微重叠（-5px 间距）

尺寸：20×32px（计分区内）
间距：-5px（重叠效果）
最大显示：4 张（超过显示"+N"）
```

## 交互设计

### 出牌流程

```
1. 玩家点击手牌
   ↓
2. 手牌向上浮起（选中态，放大 1.1 倍）
   ↓
3. 点击场牌区任意位置（或拖拽）
   ↓
4. 牌移动到目标位置
   ↓
5. 匹配判定 → 原有逻辑不变
```

### 手牌操作

| 操作 | 手势 | 效果 |
|------|------|------|
| 选中 | 单击 | 牌上浮 + 发光边框 |
| 确认出牌 | 再次单击/拖拽 | 牌飞到场区 |
| 查看详情 | 长按 | 放大显示牌面细节 |
| 整理手牌 | 滑动 | 按月份/类别排序 |

## 响应式适配

| 屏幕比例 | 布局策略 |
|----------|----------|
| 16:9（标准横屏） | 完整布局 |
| 16:10（宽屏） | 手牌区略压缩 |
| 4:3（平板） | 计分区缩小 |
| 18:9（全面屏） | 上下留黑边或扩展手牌区 |
| 竖屏模式 | 提示用户旋转设备 |
