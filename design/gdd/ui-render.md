## 牌面渲染

### 单张牌尺寸

| 参数 | 值 |
|------|-----|
| 宽度 | 60 px（移动端）/ 80 px（横屏） |
| 高度 | 96 px（移动端）/ 128 px（横屏） |
| 圆角 | 6 px |
| 手牌间距 | -15 px（重叠） |
| 场牌间距 | 8 px（不重叠） |

### 牌面状态

| 状态 | 说明 | 样式 |
|------|------|------|
| 手牌 | 玩家手中的牌 | 正面朝上，可点击 |
| 对手手牌 | 对手手中的牌 | 牌背图案 |
| 场牌 | 场上展示的牌 | 正面朝上，不可点击 |
| 计分区 | 已收集的牌 | 缩小显示，不可交互 |
| 选中 | 选中的手牌 | 上浮 10px + 发光边框 |
| 高亮 | 可与手牌配对的场牌 | 脉冲光效 |

## 动画设计

### 出牌动画

```
1. 玩家点击手牌
   ↓
2. 手牌上浮 10px（选中态）
   ↓
3. 点击确认/拖拽到场区
   ↓
4. 牌从手牌位置移动到对应场牌位置
   - 移动时间：300ms
   - 缓动函数：ease-out
   ↓
5. 如果匹配成功：
   ├─ 场牌闪烁
   ├─ 两张牌飞向计分区
   ├─ 配对提示文字："配对成功！"
   └─ 计分数字 + 动画
6. 如果匹配失败：
   └─ 牌留在场区
```

### 翻山札动画

```
1. 山札堆叠处弹出一张牌
2. 牌翻转到正面朝上
3. 移动到对应场牌位置
4. 如果匹配 → 飞向计分区
5. 如果不匹配 → 留在场区
```

## UI 组件

### 游戏主界面

```typescript
interface GameUI {
  // 区域
  opponentArea: OpponentArea;    // 对手区域
  tableArea: TableArea;          // 中央牌桌
  playerArea: PlayerArea;        // 玩家区域
  statusBar: StatusBar;          // 状态栏
  
  // 弹窗
  yakuPopup: YakuPopup | null;   // 役达成弹窗
  resultPopup: ResultPopup | null; // 结算弹窗
}

interface TableArea {
  fieldCards: CardView[];        // 场牌
  mountainPile: MountainPile;    // 山札堆叠
  capturedArea: CapturedArea;    // 计分区（可展开）
}

interface PlayerArea {
  handCards: CardView[];         // 手牌
  score: number;                 // 当前得分
  nickname: string;
  avatar: string;
  isMyTurn: boolean;
  koiKoiCount: number;           // Koi 次数显示
}

interface StatusBar {
  round: number;                 // 当前第几局
  deckRemaining: number;         // 山札剩余
  turnIndicator: string;         // "你的回合" / "对手思考中..."
  turnTimer: number | null;      // 回合倒计时（可选）
}
```

## 技术选型建议

| 技术 | 说明 |
|------|------|
| 微信小游戏 Canvas | 原生渲染，性能好 |
| Cocos Creator | 完整游戏引擎，有动画系统 |
| Egret / LayaAir | 备选游戏引擎 |
| 纯 DOM + CSS | 简单场景可用，但性能有限 |

> **推荐 MVP 使用 Cocos Creator**：有完善的动画系统、粒子效果、响应式适配，且支持微信小游戏一键发布。
