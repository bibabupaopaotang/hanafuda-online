# 花札微信小游戏 — 交互体验（UX）优化方案

## 📋 优化前问题清单

| # | 问题 | 严重程度 | 用户影响 |
|---|------|---------|---------|
| 1 | 需要"点击选中 → 再点击出牌"两步操作 | 🔴 高 | 操作繁琐，节奏拖沓 |
| 2 | 出牌无动画，瞬间切换 | 🟡 中 | 缺少视觉反馈，不够"流畅" |
| 3 | 回合提示仅为一个小色块 | 🟡 中 | 不容易注意到当前是否轮到自己 |
| 4 | 役达成弹窗使用 `wx.showModal` | 🟡 中 | 风格不统一，无法定制，两个按钮容易误触 |
| 5 | 无倒计时，无超时提示 | 🟡 中 | 不知道还剩多少时间 |
| 6 | 无触摸反馈（按下/松开） | 🟢 低 | 操作手感生硬 |
| 7 | 无拖拽支持 | 🟢 低 | 不支持更直观的出牌方式 |

## ✅ 优化后改进点

### 1. 点击即出（核心优化）

**改动前：**
```
点击手牌 → 牌上移选中 → 再次点击 → 出牌
```

**改动后：**
```
点击手牌 → 出牌动画 → 网络发送
```

- 单次触摸完成出牌，操作减少 50%
- 长按 200ms 可激活拖拽模式（可选）
- 拖拽到「场牌区」松手 = 出牌，拖到其他地方 = 取消

### 2. 出牌动画

```
手牌位置 ────抛物线弧线────▶ 场牌目标位置
           ↗ easeOutCubic ↗
           弧线高度 -30px（抛物线感）
           缩放 1 → 1.08 → 1（弹跳感）
           时长 350ms
```

- 使用 `requestAnimationFrame` 驱动，60fps
- easeOutCubic 缓动：先快后慢，符合物理直觉
- 正弦弧线：牌在飞行中有抛物线轨迹
- 微缩放：飞行中牌微微放大再缩小

### 3. 回合指示器

```
┌──────────────────┐
│  ✋ 你的回合       │ ← 绿色发光边框
│  [12s]            │ ← 倒计时（<5s变红）
│  ━━━━━━━━━       │ ← 倒计时进度条
└──────────────────┘
```

- 你的回合：绿色发光圆角矩形 + "✋ 你的回合"
- 对手回合：橙色发光 + "⏳ 对手思考中"
- 倒计时：数字 + 进度条，<5秒变红色警告
- 超时自动随机出一张

### 4. 自定义役达成弹窗

```
┌──────────────────────────────┐
│  🎉 役达成！                  │
│  ─────────────────────────   │
│  桜 + 芒 = 花见月            │
│  5分                          │
│                              │
│  [结束结算]    [こいこい]      │
└──────────────────────────────┘
```

- Canvas 内渲染，风格与游戏统一
- easeOutBack 弹跳进入动画（缩放 0.8 → 1.0）
- 淡出关闭动画（150ms）
- 两个按钮并列，颜色和文案明确

### 5. 触摸反馈

- 按下手牌：缩小 5% + 微微上抬（easeOutBack）
- 按下按钮：缩小 4% + 颜色加深
- 长按 200ms：轻微震动（`wx.vibrateShort`）

### 6. 动画循环架构

```
GameOptimized
├── flyingCards[]    → 出牌动画（requestAnimationFrame 更新）
├── modal            → 弹窗动画（进入/退出过渡）
├── turnTimer        → 倒计时更新
├── pressFeedbackId  → 按下反馈（实时动画）
└── render()         → 统一渲染入口
```

所有动画由同一个 `requestAnimationFrame` 循环驱动，避免多个定时器冲突。

## 🔧 集成方式

### 方式一：直接替换（推荐）

```bash
# 备份原版
cp packages/client/src/main.ts packages/client/src/main.ts.bak

# 替换为 JS 版
cp packages/client/src/game.js packages/client/src/main.js
# 或者在微信开发者工具中直接指向 game.js
```

### 方式二：TypeScript 集成

如果项目使用 TypeScript 编译，可以：
1. 参考 `game-optimized.ts` 的接口和类型
2. 逐步将改动合并到现有 `main.ts`

## 🎮 交互流程图

```
用户触摸屏幕
    │
    ├─ 弹窗可见？ → 关闭弹窗
    │
    ├─ 点在按钮上？ → 执行按钮回调 + 按下效果
    │
    └─ 游戏场景 + 我的回合？
        │
        ├─ 长按 200ms → 进入拖拽模式 → 拖到场牌区 → 松手出牌
        │                                     → 拖到其他地方 → 取消
        │
        └─ 短按（<200ms）→ 点击即出 → 飞行牌动画 → 网络发送
```

## ⚠️ 注意事项

1. **微信小游戏兼容性**：
   - `wx.vibrateShort` 仅在真机上有效，模拟器无效果
   - `requestAnimationFrame` 在微信小游戏环境中完全支持
   - `roundRect` 在部分旧版 Canvas API 可能需要 polyfill（已提供）

2. **网络同步**：
   - 动画播放完 **之后** 才发送网络请求
   - 如果 `state_update` 在动画期间到达，不立即渲染，等动画完成
   - 超时自动出牌是本地行为，服务器端也有超时保护

3. **性能**：
   - `requestAnimationFrame` 在动画结束后自动停止，不会空跑
   - Canvas 渲染量保持不变，只是增加了动画帧

4. **待服务器端配合**：
   - 当前超时自动出牌是纯前端行为
   - 建议服务端也实现超时逻辑（防止客户端未发请求的情况）
